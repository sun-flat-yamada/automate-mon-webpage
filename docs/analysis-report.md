# 履歴データの分析レポート

`history` ディレクトリに保存されたデータを分析し、アイテム追加（在庫追加）のタイミングに関する規則性を調査しました。

## 概要

- **分析期間**: 2026-02-01 〜 2026-02-07 (7日間)
- **総追加検知数**: 30件
- **対象ディレクトリ**: `dell_outlets_desktops`, `dell_outlets_desktops_biz`, `dell_outlets_workstations`

## 時間帯別傾向 (JST)

アイテムが追加された時間帯の分布は以下の通りです。

| 時間帯 | 検知数 | 傾向 |
| :--- | :--- | :--- |
| **00:00 - 01:00** | 2 | 深夜の日付変更直後の更新がある可能性があります。 |
| 01:00 - 10:00 | 0 | 早朝から午前中は追加が見られません。 |
| **11:00 - 12:00** | 6 | 昼前のまとまった追加が見られます。 |
| **14:00 - 15:00** | 6 | 午後早い時間の追加も見られます。 |
| **17:00 - 18:00** | 1 | 夕方に散発的な追加があります。 |
| **18:00 - 19:00** | **11** | **最大のピーク**です。夕方18時台に集中して追加される傾向があります。 |
| **20:00 - 21:00** | 4 | 夜間にも追加が行われています。 |
| 21:00 - 24:00 | 0 | 深くは追加が見られません。 |

## 曜日別傾向 (JST)

- **水・木・金・日曜日** に追加が集中しています。
- 火曜日は今回の期間中では追加がありませんでした。
- 土曜日・月曜日は件数が少ない傾向にあります。

## 結論と推奨

- **18:00 (JST)** 前後が最も在庫追加の可能性が高い時間帯です。
- 次いで **11:00**、**14:00**、**20:00** が狙い目です。
- 監視頻度を調整する場合、**早朝 (01:00 - 10:00)** は頻度を下げ、**夕方 (17:00 - 21:00)** および **11:00、14:00** 前後は頻度を上げると効率的かもしれません。
